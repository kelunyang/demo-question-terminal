# 台灣的族群政治 — UI 改版規格（specupgrade.md）

> 本文件只處理**前端呈現形式**的改版，不動 `spec.md` 已定案的資料模型／後端函式／AI prompt 邏輯。
> 目的：把現有「三段式版面（年度／欄位／圖表類型 el-segmented ＋ D3 圖 ＋ 描述／出題／批改各自獨立區塊）」
> 整個改成**單一對話式終端機（terminal log）**的呈現典範，用於對外 demo，讓使用者（設定為「老師」，
> 評估這是不是一個能用的教學工具）像在跟一個 AI 助教打交道一樣，體驗「選資料 → 出圖 → AI 分析 →
> 出題 → 作答 → 批改」的完整流程。

## 1. 改版目標與情境

- **場景**：對外 demo，觀眾角色設定為**老師**，評估這個網站能不能當教學輔助工具。
- **典範轉移**：整頁不再有獨立的年度/欄位選單區、圖表區、描述區、出題 drawer——全部收進**同一條由上而下累積的對話 log**，訊息只會往下追加、不覆寫，老師可以往上滑看到完整操作歷程。
- **視覺惡趣味**：整體 UI 比照**老式 IBM 終端機**（CRT 掃描線、磷光色、閃爍游標、等寬 webfont），但排版與互動邏輯**模仿 Claude Code CLI**（線性 log、底部固定輸入框、結構化內容用邊框區塊呈現，而非聊天氣泡）——復古機殼、現代 agent 互動節奏的混搭。
- **兩種資料選取路徑並存**：老師可以（a）直接打字跟 AI 聊，讓 Insight 助手用 tool use 幫他選年度/欄位/篩選（真呼叫 API）；也可以（b）用終端機 log 裡嵌入的原生 UI 元件（年度按鈕列、欄位勾選列）自己選（純前端資料函式，不叫 LLM）。兩條路徑最終都在同一條 log 上長出一個圖表區塊。
- **不是全部都要真的呼叫 LLM**：只有「需要語意理解／生成文字」的環節才叫真的 DeepSeek API；老師自己點 UI 選單選年度/欄位、選題型按鈕，都是直接呼叫既有資料函式或送出結構化參數，跟現在的行為完全一樣，不需要也不應該假裝成一次 AI 對話輪次去消耗 API 呼叫。

## 2. 視覺風格：終端機 × Claude Code 混合美學

| 項目 | 規格 |
|---|---|
| 配色 | 經典磷光配色，預設**磷光綠**（前景 `#33ff33`／背景 `#0a0a0a`），日後可加設定切換成琥珀 `#ffb000`（見待確認事項） |
| 字型 | 等寬 webfont，例如 `IBM Plex Mono` 或 `VT323`／`Share Tech Mono`（Google Fonts CDN 引入即可，與現有 Font Awesome CDN 引用方式一致） |
| 掃描線 | 全螢幕疊一層 `repeating-linear-gradient` 橫線紋理＋極輕微 flicker/vignette 動畫，`pointer-events:none`，不影響底下互動 |
| 閃爍游標 | 底部輸入行尾端一個實心方塊字元，`blink 1s step-end infinite`；AI 打字機輸出時，游標跟著文字尾端一起移動，強化「即時輸出」的終端機感 |
| 版面邏輯（仿 Claude Code） | **不用聊天氣泡**、不做左右對齊，改成線性 log：使用者輸入永遠以提示符號開頭（如 `❯ `）；AI／系統輸出純文字左靠齊，前面掛一個角色標記；結構化內容（圖表、選單、出題表單、批改結果）用 box-drawing 風格邊框包起來，標題列標明區塊內容（例如 `┌─ 圖表：2025／統獨立場 ─┐`），呼應 Claude Code 呈現工具結果／diff 的邊框卡片感 |
| **真假 AI 輸出的標記區別** | 真的呼叫 DeepSeek 產生的內容（S2(a) Insight 對話、S5 分析、S7c 審題、S8 出題(AI 出的分支)、S9/S12 評論重出、S11 批改）前綴用 **🤖**；純前端寫死、沒呼叫 API 的系統提示／選單（S0 開場腳本、S1/S6/S9/S15 的選單文字、S7a/S7b/S13 的提示句與表單欄位等）維持用 **●**。兩者都走一樣的打字機效果與線性 log 版面，只差這一個 emoji 前綴——老師事後回看時，看前綴就知道哪句是 AI 真的想出來的、哪句只是網站固定的選單文字 |
| 多選項互動（仿 Claude Code 權限選單） | 所有「二選一／多選一」的節點（模式選擇、出題後選擇、批改後選擇、結束選單…）都用**編號選單**呈現，例如：<br>`[1] 跟我聊`<br>`[2] 我自己選`<br>`❯ _`<br>可以滑鼠點選項，也可以直接打數字＋Enter，兩種操作都成立 |
| 輸入框狀態 | 固定在畫面最下方；**只有在流程明確需要老師輸入時才啟用**（見第 3 節各 S 節點），其餘時間顯示為灰階 disabled／placeholder 提示「請由上方選單操作」，避免老師亂打字打斷流程 |
| D3 圖表配色 | 建議**保留全彩**，不強制單色化——在滿版磷光文字終端機之間，全彩圖表形成「文字 vs 圖形」的視覺焦點反差，比整頁單色更有 demo 效果（開放討論，見待確認事項） |
| AI 思考流呈現 | 每次真呼叫 DeepSeek（S5 分析／S7c 審題／S8 出題(AI 出的分支)／S9 評論重出／S11 批改／S12 評論重出，S2(a) 若保留 Insight 對話亦同）都會拿到 `reasoning_content`：**呼叫進行中**，思考流即時用打字機效果印在終端機上，樣式跟正式輸出區分（例如較暗的灰階／斜體，或加個 `思考中…` 的摺疊框樣式）；**呼叫結束、拿到最終內容後**，思考流自動摺疊成一行摘要（例如 `▸ 思考過程（點此展開）`），底下才接著印出 DeepSeek 的最終輸出；老師隨時可以點那一行展開/收合看完整思考過程，收合不影響已印出的最終結果。思考流**只在前端呈現，不寫進 `RESULTSHEET_ID`**——第 5 節存檔規格裡的 `analysisresult`/`question`/`airesponse` 都只存最終輸出內容，不含思考過程 |

### 2.1 物理彩蛋清單（Matter.js headless 驅動）

延伸自「新訊息插入時舊訊息被頂一下」那套彈簧系統——全部用 **Matter.js 純數值模擬（不掛 `Matter.Render`／不用 canvas）**，每個 `requestAnimationFrame` 讀 body 位置寫回 DOM 元素的 `transform`，本質上都是同一套引擎、套在不同觸發點上的花樣。全部遵守 `prefers-reduced-motion`（設定減少動態效果的裝置直接跳過，退回無動畫的陽春版本），CDN 載入失敗也不影響核心功能。

| 觸發時機 | 效果 |
|---|---|
| S15 選「是，重新對話」 | 整條 timeline 裡每一則訊息依序失去支撐、依序往下墜落消失，畫面清空後才印出下一套（隨機抽選的）開場白 |
| S9／S12 老師點「請 AI 重新來一次」 | 目前顯示的舊題目／舊批改結果區塊像被揉成一團，帶旋轉往旁邊飛出畫面；新結果稍後從上方落下，輕彈一下才定位 |
| 新圖表區塊出現在 timeline 上 | 掉落到定位時輕輕彈跳一下才靜止（跟「新訊息頂舊訊息」共用同一套彈簧參數） |
| S14 老師送出 email | 一個小紙飛機圖示（✈️）帶拋物線軌跡飛出畫面，作為「資料已寄出」的視覺回饋 |
| S12 `el-rate` 評分結果 | 打滿 5 顆 😄：整排圖示輕輕跳動像歡呼；打 1 顆 😭：圖示微微下垂／震動一下，給點情緒回饋（純視覺，不影響 `humanrate` 存檔數值） |
| 老師打字時的閃爍游標 | 游標追字尾不是死板瞬移，而是帶一點輕微彈簧慣性、稍微「追趕」文字尾端，強化終端機手感 |
| AI 呼叫等待中（S2a／S5／S7c／S8(AI出的分支)／S9重出／S11／S12重出） | 等待動畫換成 2–3 個小方塊像思考泡泡一樣上下彈跳，取代單純的 spinner 轉圈 |

## 3. 完整對話流程（老師視角，狀態機）

流程中「是否真呼叫 API」欄位標明每個節點有沒有真的打到 DeepSeek 或資料後端；沒有標的節點是純前端狀態切換／印字。

### 3.0 終端機開機動畫（頁面首次載入的前導，先於 S0）

進 S0 開場白之前，先播一段模擬老終端機開機的動畫，順便掩護第一次真正的資料載入（`listAvailableYears()`），不是純表演：

1. 螢幕由暗到亮，帶一點輕微的消磁波紋（畫面短暫扭曲/色偏再穩定）
2. 中央一個亮點快速橫向、再縱向展開至滿版——經典 CRT 開機視覺記憶點
3. 掃描線由上而下刷過畫面一次
4. BIOS／POST 風格文字逐行印出：
   ```
   TNSS TERMINAL BIOS v1.0
   MEMORY CHECK.......... OK
   LOADING DATASET (16 YEARS)... OK
   ESTABLISHING LINK TO DEEPSEEK... OK
   ALL SYSTEMS NOMINAL
   ```
   其中 `LOADING DATASET` 那一行**真的等 `listAvailableYears()` 回應**才印出 `OK`，把開機動畫跟第一次資料載入合而為一，不用另外做 loading spinner；**設 3 秒上限**，超過就先印 `OK` 讓動畫跑完、資料在背景繼續處理，避免 GAS 一時慢卡住整個開場
5. 亮度收尾定住，游標開始閃爍、`❯` 提示符號浮現，接著才進入 **S0**（3.1 節開場腳本池，隨機挑一套打字機播放）

**S15 選「重新對話」回到 S0 時，只播簡化版**：畫面閃爍 2–3 下＋掃描線刷一次，不重播完整 BIOS 文字（避免同一場 demo 反覆看到一樣的開機畫面拖節奏）。

全程遵守 `prefers-reduced-motion`（直接跳過動畫），並且**可以點擊/觸碰畫面任一處跳過**，方便 demo 現場控制節奏、對不同觀眾重複展示時不用每次乾等。

### 3.1 開場腳本池（S0，隨機挑一套）

同一批文案調性（中性、親民，不點名統獨/認同等具體立場議題），每次進入 S0（含 S15 選「重新對話」後回到 S0）**從下面幾套裡隨機抽一套**播放，避免同一場 demo 重跑好幾次時老是講一模一樣的話。全部是前端寫死的純腳本，不呼叫任何 API。腳本 1–5 平均分配機率；腳本 6 是隱藏彩蛋，抓低機率（例如 1%）獨立抽中，不告知使用者這是彩蛋。

**腳本 1**
```
● 嗨，我是這個網站的 AI 助教。
● 先自我介紹一下這裡在做什麼：我們手上有台灣國家安全調查（TNSS）16 年份的民調資料，
  從本省閩南、客家、外省到原住民都有涵蓋。
● 你可以把這裡想成一個提問工具——想知道不同族群、不同世代的台灣人，
  對台灣社會的一些重要議題各自怎麼想，都可以從這份資料裡挖出來看。
● 資料我已經幫你準備好了，隨時可以開始。
```

**腳本 2**
```
● 哈囉，歡迎來到「台灣的族群政治」。
● 這裡的資料來自台灣國家安全調查（TNSS），跨了 16 個年度，
  涵蓋本省閩南、客家、外省、原住民等不同族群的受訪者。
● 簡單說，這是一個讓你自己動手挖資料的地方：選個年度、選個議題，
  就能看到不同族群、不同世代的台灣人怎麼想。
● 功課我已經幫你做完了——資料都下載整理好了，我們可以直接開始。
```

**腳本 3**
```
● 你好，我是負責帶你逛這份資料的 AI 助教。
● 這份資料庫記錄了台灣過去 16 個年度的民意調查，受訪者橫跨各個族群與世代。
● 你可以把我們接下來要做的事，想成是在問資料一個個問題：
  「這個世代的人怎麼看？」「不同族群之間差在哪裡？」
```

```
● 嗨！歡迎使用「台灣的族群政治」資料探索工具。
● 背後的資料是 Duke 大學做的台灣國家安全調查，累積了 16 年份，
  受訪者包含本省閩南、客家、外省、原住民等各種族群背景。
● 等一下你可以自己選資料，也可以直接跟我聊、讓我幫你找——
  目標都是同一個：看看不同族群、不同世代的台灣人，關心的重點有什麼不一樣。
● 資料已經在這裡準備好了，我們開始吧。
```

**腳本 5**
```
● 哈囉，我是這裡的 AI 小幫手。
● 先講講背景：這是一份跨 16 個年度的全國性民意調查（TNSS），
  受訪者組成涵蓋台灣各個族群，時間跨度也夠長，可以看世代差異。
● 你等一下要做的事，其實就是挑一個角度切進去看——
  某個族群、某個世代，對某些議題的想法。

```

**腳本 6（隱藏版彩蛋，非物理效果——抽中機率抓 1%，不會告知使用者這是彩蛋）**
```
● （清清嗓子）哈囉，資料介紹我已經講了不知道第幾次了，今天講簡短版。
● 台灣國家安全調查，16 年份，各族群都有涵蓋，族繁不及備載。

```

### 3.2 其他節點的提示文案池（隨機挑一句，語氣詼諧化）

跟 3.1 開場腳本池同樣的邏輯：這些節點原本都只有一句死板的固定文案，現在每個都給 2–3 句語氣輕鬆一點的版本，每次觸發隨機挑一句，避免整場對話讀起來太呆板、太像制式系統訊息。這些都是 **●**（純前端寫死，不呼叫 API）的系統提示，選單本身的編號選項文字不變，只有「帶出選單前面那句話」隨機變化。

**S1 模式選擇**
```
● 你想怎麼開始？跟我聊，還是你想自己動手？兩種我都奉陪。
● 選個開場方式吧——用嘴巴指揮我，還是親自下海選？
● 第一步：你要當甩手掌櫃（跟我聊），還是親自操刀（自己選）？
```

**S6 分析後選擇**
```
● 分析看完啦，接下來呢？想考考學生，還是再多聊幾句？
● 這張圖先分析到這——要出題驗收一下，還是我們再多挖一點？
● 分析告一段落，準備出題了嗎？還是這張圖你還沒問夠？
```

**S7a／S7b 出題準備（輸入框 placeholder）**
```
跟我說說想考什麼，或者直接挑題型讓我自己看著辦
出題方向隨便你，不寫的話我就自由發揮了
想好考什麼了嗎？沒想法也沒關係，選個題型我來扛
```

**S9 出題後選擇（AI 出的題）**
```
● 這題還滿意嗎？不滿意我可以重寫，滿意就直接開考。
● 題目出好了，挑剔一下也行，直接應戰也行，你決定。
● 看看這題順不順眼——嫌棄的話我重寫，順眼就開始作答。
```

**S9 出題後選擇（老師自己出的題）**
```
● 題目跟審題建議都在上面了，要不要再出一題，還是直接開始作答？
● 你的題目我審完了，再來一題，還是這樣就夠、直接考？
● 審題建議寫好了，看你要不要再加一題，還是準備開戰。
```

**S13 詢問留 email**
```
● 都跑完一輪了，願意留個 email 讓我們知道這份資料還幫上了誰的忙嗎？
● 最後一件事：想留個信箱給我們做個記錄嗎？不留也沒關係。
● 如果不嫌麻煩，留個 email 讓我們知道這工具有被用過（不留也行）。
```

**S15 結束選單**
```
● 這輪就到這裡，要不要重新來一次，換張圖再玩一輪？
● 對話結束啦，要我把畫面清空重新開始嗎？
● 跑完一整趟了，辛苦了，要不要重新開一局？
```

> 範圍說明：S12「重新給意見，請 AI 再批改一次」那顆按鈕是功能性按鈕標籤，不是「AI 開口說話」的敘事文字，這次沒有一併詼諧化；如果之後也想讓它有點個性，可以比照這裡的做法另外補文案池。

| 階段 | 終端機呈現 | 老師可做的事／觸發方式 | 真呼叫 API？ |
|---|---|---|---|

## 9. Worker 濫用防護（Origin 白名單 + Rate Limit）

> 2026-07 實作。動機、Origin 可偽造的限制與踩坑細節見 `issue.md` 第 11 條，本節只記「最終規格」。

**威脅模型**：前端直連 Worker 的共用 token（`APP_SHARED_TOKEN`）寫在公開的 GAS 網頁裡，`view-source` 就拿得到，**不是真正的機密**。因此無法「擋死」，策略是「擋掉爬蟲／隨手濫用，並把最壞傷害鎖住」。金鑰（`DEEPSEEK_API_KEY`）本身仍只在 Worker secret，前端與 GAS 都拿不到。

**兩道防線（都在 `worker/src/index.js`）**：

1. **Origin 白名單**：只放行 `Origin` 字尾屬於 `ALLOWED_ORIGIN_SUFFIXES`（`.googleusercontent.com`、`script.google.com`）的請求，其餘回 `403 Forbidden origin`。
   - GAS HtmlService 沙箱 iframe 的實際 Origin 是 `https://n-<會浮動的 hash>-0lu-script.googleusercontent.com`（已實測），**hash 每次可能不同 → 必須用 `endsWith` 字尾比對，不可寫死完整網域**。
   - 用 `ENFORCE_ORIGIN` 布林旗標控制：`false`＝只 `console.log` 記錄不阻擋（用來先觀察真實 Origin，避免設錯把 app 打掛）；確認後改 `true` 才真正阻擋。目前為 `true`。
   - CORS 一併從 `Access-Control-Allow-Origin: *` 收成「只把白名單內的 Origin 原樣 echo 回去」＋ `Vary: Origin`；非白名單不帶 ACAO（瀏覽器端讀不到回應）。`corsHeaders(origin)`／`jsonResponse(obj,status,origin)` 都要把 origin 串進去（含 OPTIONS 預檢、SSE 串流、錯誤回應各出口）。

2. **每來源 IP Rate Limit**：Cloudflare Workers 原生 rate-limit binding（`wrangler.toml` 的 `[[ratelimits]]` + `[ratelimits.simple]`，`name = "RATE_LIMITER"`），**30 次 / 60 秒**（`period` 只能是 `10` 或 `60`；需 wrangler ≥ 4.36）。key 用 `CF-Connecting-IP`。超限回 `429`。
   - 程式端 `underRateLimit()` 設計成 **fail-open**：`env.RATE_LIMITER` 不存在或 `.limit()` 拋錯時一律放行，避免限流器本身把正常流量擋死。
   - 放在 token 檢查「之後」，連「拿到 token 想狂打」也一起限速。

**殘留缺口（本防護擋不住的）＋建議補強（非本 repo 內設定）**：

- `Origin` header **非瀏覽器 client（curl／腳本）可偽造** → 有心人複製 token 又偽造 Origin，上面兩道都擋不住，只能靠 rate limit 拖慢。
- 真正能「把最壞金額鎖死」的地板是 **DeepSeek 帳號後台的消費上限**（跟前端／GAS／Worker 都無關，純在 DeepSeek 官網設數字）——**建議務必設定**。
- 選配：Cloudflare 儀表板的 **Bot Fight Mode**（免費方案即有的一鍵開關）。
- 不採用 **Cloudflare Turnstile**：GAS 沙箱是浮動的 `*.googleusercontent.com` origin ＋ 需在 Google 沙箱 iframe 內再嵌 `challenges.cloudflare.com` 跨網域 iframe（受第三方 storage 分割與沙箱 CSP 影響），屬「勉強能通、易壞、不受官方支持」，不值得為 demo 投入。若哪天真要「只放行真人」，正解是把前端從 GAS 搬到 Cloudflare Pages 這類真正的靜態站，Turnstile 在那邊才原生順暢。
